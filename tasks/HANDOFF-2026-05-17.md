   - "Try it yourself" suggestions populated by L2+ templates in
     `explain.c`, completed when the cursor is past the first word
     and not in a file-command context.
   - Quoted and unquoted filename completion for `load`, `read`,
     `dump`, `dumpnative` (quoted form strips the `"`, completes
     against `rl_filename_completion_function`, restores the
     `"`, and sets `rl_completion_append_character = '"'` for the
     closing quote on a unique match).
   - Label completion for `breakpoint` and `delete`, driven by a
     new public `for_each_label` iterator in `src/sym-tbl.c`.
3. **Man page wired into install** — `Documentation/spim.man`
   renamed to `Documentation/spim.1` (meson's `install_man`
   requires the section-number filename extension); `meson.build`
   gains `install_man('Documentation/spim.1')`. Dockerfile now
   copies `Documentation/` into the image. Result:
   `$prefix/share/man/man1/spim.1`.
4. **Five-level `-explain` ladder**: replace `bool explain_mode`
   with `int explain_level`. CLI accepts `-explain=N` and
   `-explain N` (and `-x N`) for N in 0..4. Bare `-explain` = 3
   (back-compat with prior behavior). See "Level contract" below.
5. **Post-execute narration refactor**: `explain_before` shrank to
   "snapshot only" (registers + memory for stores via new
   `snap_has_mem` / `snap_mem_addr` / `snap_mem_val`). The
   `switch (OPCODE(inst))` dispatch and all templates moved to
   `explain_after`, run *after* execution, so every line of output
   describes something that has already happened by the time the
   user sees it. Wording throughout is past tense ("computed",
   "wrote", "shifted") and inputs read from `snap_R[]` while
   outputs read from live `R[]`.
6. **Unicode box-drawing** — all seven bit-layout renderers (three
   L3 + four L4) and the `hr()` section divider now use
   `┌─┬─┐ │ └─┴─┘ ─` instead of `+ - |`. Same column math, cleaner
   visual.
7. **Stepped header + one-time legend** — the per-step header in
   `explain_after` is a labeled multi-line block:
   ```
   Stepped at PC = 0x00400024:
       memory[0x00400024] = 0x34020004   →   ori $v0, $r0, 4
       source line 40: li $v0, 4
   ```
   Plus the pseudo-op header underneath when applicable. A
   once-per-session legend at the top of explain mode explains
   what each piece of the header means. Extracted into a public
   `explain_print_step_header(pc, inst)`.
8. **L4 progressive bit-field decoding** — replaces the single L3
   box with 2–3 frames of the same box redrawn progressively as
   the CPU's hierarchical decoder resolves the word
   (opcode → funct or rt-as-selector → remaining fields).
   Undecoded fields show `?` placeholders both in the box and in
   the label/value rows.
9. **L2+ category + modifier preamble** — every L2+ narration
   leads with `Category: NAME` (one of Arithmetic, Logical, Data
   transfer, Conditional branch, Unconditional jump, System) and
   `Modifiers in \`mnemonic\`:` describing each suffix (`u`, `i`,
   `v`, `al`, `w`, `h`, `b`). First time each category or
   modifier appears in the session, the full multi-line
   description is emitted; subsequent appearances get the short
   form. Categorization follows COD 4e Green Sheet.
10. **L1 trimmed to just header + pseudo-op** — at level 1 the
    student sees the labeled header (same four data items as the
    L0 compact one-liner, just relayed out) plus the pseudo-op
    block when applicable. No `What it did:`, no Inputs/Wrote,
    no Try-it-yourself, no PC delta — those are L2+.

## Gotchas worth knowing

**libedit ≠ GNU readline**, two ways:

1. Both `rl_completer_word_break_characters` and
   `rl_basic_word_break_characters` must be set, or libedit
   treats `$` as a word break and `print $`+Tab corrupts to
   `print $$`. Both are set to `" \t\n"` in `top_level()`.
2. libedit doesn't auto-fall-back to filename completion when
   `rl_attempted_completion_function` returns NULL — Tab just
   gets passed through as a literal character. Must call
   `rl_filename_completion_function` explicitly. Same for any
   other "delegate to the default completer" intent.

Both are explained in inline comments in `src/spim.c`.

**ENCODING(inst) was historically broken**, fixed in `include/op.h`:
the X-macro `OP(...)` list used to be wrapped in an `#ifndef OP_H`
guard, so `inst.c` (which includes `op.h` three times to build
three tables) had two of three tables silently empty. The guard
was scoped down to just the type-macro definitions
(`OP_H_TYPES`). All three tables populate now; `inst_encode()`
returns the real binary word.

**Pseudo-op continuation tracking**: a multi-instruction pseudo-op
like `la $t7, dst` → `lui $at, ...` + `ori $rd, $at, ...` only
attaches a `source_line` to the first emitted instruction. The
existing `pending_pseudo_name` / `pending_pseudo_multi` static
flags in `src/explain.c` handle the continuation-hint emission
on the second instruction; if any new template family is added
that can come from a multi-inst pseudo-op, make sure the
continuation flag survives across its emit path.

**libedit completion with quoted filenames** uses
`rl_completion_append_character = '"'` — that's a per-call
override and could leak across completion attempts. The
`spim_completion` dispatcher resets it to `' '` at the top of
every call so subsequent non-quote completions don't pick up the
override.
