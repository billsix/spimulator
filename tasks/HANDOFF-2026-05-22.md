## Strand 1 ‚Äî Portability cleanup (master branch, landed)

Per the May session that started this work, the goal was to strip
dead C portability code Bill no longer wanted to maintain.  Five
phases landed in sequence on master:

- **Phase 1 ‚Äî Drop Windows.**  ~200 lines removed across `src/spim.c`,
  `src/syscall.c` (the `myInvalidParameterHandler` Watson-disable
  hack and `_open`/`_read`/`_write`/`_close` aliases),
  `src/run.c` (`CreateWaitableTimer`/`SleepEx`/`Windows.h`),
  `src/string-stream.c` (`_vsnprintf` branch),
  `src/spim-utils.c` (`_MSC_VER` `_environ` shim), and
  `meson.build` (the MSVC-as-C++98 hack + Windows link_libraries).

- **Phase 2 (modified) ‚Äî Big-endian via meson auto-detect.**
  Kept `MIPS1` per Bill's call.  Added a 4-line block in
  `meson.build` that uses `host_machine.endian()` to set
  `-DSPIM_BIGENDIAN` automatically when needed.  `spim.h`'s
  comment now points to meson instead of the long-gone Makefile.

- **Phase 3 ‚Äî `int32`/`uint32` ‚Üí `<stdint.h>`.**  Deleted the
  `typedef int int32` / `typedef unsigned int uint32` from
  `spim.h`; added `#include <stdint.h>`; simplified `SIGN_EX(X)`
  from a 4-line bit-twiddle to `(int32_t)(int16_t)(X)`; bulk
  sed-rename of ~95 occurrences across 12 files; `mem_word` and
  `mem_addr` now use stdint types; splint `/*@alt ‚Ä¶@*/` annotations
  dropped.

- **Phase 4 ‚Äî Dead C compatibility shims.**  Deleted
  `NEED_MEM_FUNCTIONS`/`memclr` macro + 8 `memclr` call sites
  rewritten to `memset(p, 0, n)`.  Deleted `NEED_VFPRINTF`
  blocks across `error`/`fatal_error`/`run_error`/`write_output`
  (5 sites).  Deleted the K&R-style `NEED_VSPRINTF` `vsprintf`
  reaching into `FILE*` internals, plus broken `NEED_STRTOL` and
  `NEED_STRTOUL` reimplementations.

- **Phase 5 ‚Äî Polish.**  Removed `QSORT_FUNC` macro by giving
  `compare_pair_value` the standard `(const void*, const void*)`
  signature.  Removed `#ifdef mips`/`<nan.h>` IRIX block in
  `run.c`.  Investigated and **kept** `intptr_union` (70 use
  sites legitimately model a sum type) and `streq` (a legit
  local idiom).

All 17 regression tests stayed green throughout.  This is all
in the merged history on master.

## Strand 3 ‚Äî AST migration (ast branch, in progress, NOT MERGED)

Work lives on the `ast` branch (`6b38822` is the head).  20 commits
on top of master.  Migrates spim from syntax-directed translation
to an AST representation, keeping SDT alive as a parallel path
behind `-parser=sdt`.

What landed:

- **Phase 1 ‚Äî Event log.**  `include/asm_event.h` + `src/asm_event.c`
  with an observer pattern that fires events as the assembler runs.
  Instrumentation in `inst.c`/`data.c`/`sym-tbl.c` at every action
  site.  `-listing FILE` flag prints a human-readable trace.

- **Phase 2a ‚Äî Action-site catalog.**  `tasks/parse-actions-catalog.md`
  documenting all 167 state-mutating call sites in `parser.c` and
  what AST node would absorb each.

- **Phase 2b‚Äì2c ‚Äî AST types + constructors.**  `include/ast.h`
  with 23 node kinds (instruction families, data directives,
  segment directives, labels, file root, error sentinel).
  `src/ast.c` with constructors, recursive `ast_free`, indented
  `ast_print`, JSON `ast_print_json`.

- **Phase 2d ‚Äî Parser produces AST.**  Dispatch helpers
  (`emit_r`, `emit_i`, etc.) in `parser.c` route calls to either
  inline emit (SDT) or AST construction (AST mode).  167 action
  call sites renamed.  `-parser=ast` flag plus `-print-ast` /
  `-print-ast-json` / `-show-expansion` flags that skip the emit
  pass to dump the tree.

- **Phase 2e ‚Äî emit_ast walker.**  In `parser.c` at the bottom.
  Walks the file in source order, calling the action helpers with
  per-node `line_no` restored.  Side-channel forward-ref handling
  for `.word LABEL` cases.

- **Phase 2f ‚Äî Switch AST mode to deferred + fix pseudo_op.c
  bypass.**  Exposed `emit_*` helpers via `parser.h`.  Routed all
  ~14 direct action-helper calls in `pseudo_op.c` through the
  dispatch wrappers.  `defer_emit_to_emit_ast = true` is the
  default; AST mode now drives via `emit_ast`, no inline emit.

- **Phase 3 ‚Äî `AST_PSEUDO` wraps pseudo-op expansion.**  Added
  `current_pseudo` + `ast_append` helper.  `parse_pseudo` wraps
  its expansion in an `AST_PSEUDO` node carrying the original
  mnemonic.  `-show-expansion` prints just the pseudo wrappers.
  Note: this captures *parser-level* expansion (`la` ‚Üí `ori`),
  not *encoder-level* expansion (`la` ‚Üí `lui + ori` when the
  address doesn't fit 16 bits ‚Äî that happens deeper inside
  `i_type_inst` and bypasses the AST).

- **Phase 4 ‚Äî Teaching surfaces + GUI bridge.**  `-print-ast-json`
  flag emits a clean JSON tree to stdout.  Validated as parseable
  JSON.  `Documentation/spim.1` updated with man-page entries for
  every new flag.  `/examples/TEACHING-ASSEMBLER-INTERNALS.md`
  walks a student through the four teaching surfaces (`-print-ast`,
  `-show-expansion`, `-listing`, `-explain`) on a tiny hello.asm.

- **Phase 6 ‚Äî Comment hygiene.**  Stripped "Phase N" language
  from `include/parser.h`, `include/ast.h`, `src/ast.c`,
  `src/parser.c`.  Replaced with intent-level commentary.  Moved
  6 migration plan docs into `tasks/archive/`.  Added
  `tasks/archive/README.md` explaining what's there.

- **Parity test added.**  `ast_parity_all` runs every
  representative `.s` file (20 programs) through both
  `-parser=sdt` and `-parser=ast`, diffs the text + data dumps.
  Runs in ~0.33s.  Catches drift between the two parsers.
  Companion to the existing single-program `ast_parity` test.

The default parser mode is still SDT (PARSE_DIRECT).  The
AST-inspecting flags auto-flip to AST mode; explicit
`-parser=ast` runs the AST path end-to-end.

Status: **22 regression tests, all passing.**  The `core` test
times out intermittently under suite-level parallelism (~1
in every few runs); passes alone every time.  Pre-existing
flakiness, not introduced by the AST work.
